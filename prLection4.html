УП.05 — Разработка модели архитектуры ИС на примере "Интернет-магазин"

Для ознакомления со списком лекций, перейдите по ссылке.

Цель

Спроектировать и отобразить графически основные компоненты системы, их взаимодействие и место в общей структуре.

1. Выбор типа архитектуры

Для веб-приложений, таких как интернет-магазин, наиболее распространена трехуровневая (трехзвенная) архитектура. Она обеспечивает модульность, масштабируемость и безопасность.

  • Уровень 1 (Presentation Layer): Клиентская часть (браузер).
  • Уровень 2 (Business Logic Layer): Серверная часть — логика системы.
  • Уровень 3 (Data Access Layer): Уровень данных — хранение информации.

2. Декомпозиция архитектуры на модели (диаграммы)

Создаем несколько взаимосвязанных моделей:

Модель 1: Контекстная диаграмма (Диаграмма окружения)

Показать взаимодействие системы с внешними системами и субъектами:

  • Покупатель: взаимодействует через браузер.
  • Платежный шлюз: внешняя система для платежей (например, ЮKassa).
  • Курьерская служба: передача данных о доставке, получение трек-номера.
  • Админ магазина: управление контентом и заказами через админ-панель.

Модель 2: Компонентная диаграмма (Логическая архитектура)

Основные программные модули и связи между ними:

  • Frontend: интерфейс пользователя (React, Vue.js).
  • Backend (сервер):
    • API Gateway — точка входа.
    • Модуль аутентификации.
    • Модуль каталога.
    • Модуль заказов.
    • Модуль оплаты.
    • Модуль уведомлений.
  • Уровень данных:
    • База данных (PostgreSQL/MySQL).
    • Кеш (Redis).
    • Файловое хранилище (S3).
